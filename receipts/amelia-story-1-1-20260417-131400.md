## Implementation Details

### useExpenses Hook
- **Initialization:** Loads expenses from localStorage on mount
- **Auto-save:** useEffect watches expenses array, saves to localStorage on change
- **addExpense():** Validates amount > 0, category set, creates ID (timestamp), returns new expense
- **getRecentExpenses(limit):** Returns sorted by date descending
- **deleteExpense(id):** Removes expense by ID
- **getTotalByCategory():** Returns object with category totals

### AddExpenseForm Component
- **State:** amount, category, date, note, error
- **Validation:** Client-side before onSubmit
- **UX:** 
  - Error messages displayed prominently (red box)
  - Loading state: disabled form + "Adding..." button text during submission
  - Required fields marked with red asterisk
  - Placeholder text for guidance
- **Accessibility:** Form labels with htmlFor, input names

### App Integration
- **State management:** useExpenses hook + local isSubmitting state
- **Event handler:** handleAddExpense wraps addExpense with loading state
- **Layout:** Responsive grid (1 col mobile, 3 col desktop)
  - Left: AddExpenseForm
  - Right: Dashboard + ExpenseList
