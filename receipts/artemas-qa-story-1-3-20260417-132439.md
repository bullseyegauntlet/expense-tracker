### ✅ AC 2: Edit form pre-populates with current expense data

  - `setCategory(expense.category)` — selects current category
  - `setDate(expense.date)` — fills date field

  - Error state cleared on pre-population
- **PASS** ✅

### ✅ AC 3: User can change amount, category, date, or note
**Implementation verified:**
- Amount field: `<input type="number" ... onChange={(e) => setAmount(e.target.value)} />` (lines 74-82)
- Category field: `<select ... onChange={(e) => setCategory(e.target.value)} />` (lines 90-100)
- Date field: `<input type="date" ... onChange={(e) => setDate(e.target.value)} />` (lines 108-114)
- Note field: `<textarea ... onChange={(e) => setNote(e.target.value)} />` (lines 122-131)
- All fields properly bound to state and have onChange handlers
- Step attribute on amount: `step="0.01"` allows decimal editing
- Min constraint: `min="0"` prevents negative amounts in UI (validation in backend)
- **PASS** ✅

### ✅ AC 4: Clicking "Save" validates input (amount > 0, category required)
**Implementation verified in EditExpenseForm:**
- handleSubmit validation (lines 24-39):
  - Amount check: `!amount || parseFloat(amount) <= 0` → error "Amount must be greater than 0"
  - Category check: `!category || category.trim() === ''` → error "Category must be selected"
- Error state display (lines 65-69): Red banner with error message
- Form submission prevented if validation fails
- **Also verified in useExpenses.js updateExpense():**
  - Identical validation logic (lines 59-69 in useExpenses.js)
  - Validates before state update
  - Throws error if validation fails (caught by try/catch in App)
- **PASS** ✅

### ✅ AC 5: On success, expense updates in list and dashboard

  - setExpenses triggers state update
- localStorage auto-save: useEffect (lines 12-14 in useExpenses.js)
  - Watches `expenses` dependency
  - Persists updated array to localStorage after state change
- Dashboard & ExpenseList re-render:
  - Both components receive data from App via props
  - Dashboard uses `totalByCategory` from getTotalByCategory() (recalculated)
  - ExpenseList uses `recentExpenses` from getRecentExpenses() (sorted, updated)
- **PASS** ✅
