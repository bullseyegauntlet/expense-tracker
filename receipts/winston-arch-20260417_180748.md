## Notes for Implementation Team

**Critical Success Factors:**
1. **Lazy load Recharts** — Don't import at top level; import inside ChartDisplay component
2. **Debounce localStorage writes** — Critical for responsiveness with frequent updates
3. **Memoize monthly calculations** — useMemo(() => {...}, [expenses, selectedMonth])
4. **Test on real mobile device** — Simulators miss touch/input quirks
5. **Validate CSV export** — Open in Excel + Google Sheets before release

**Common Pitfalls to Avoid:**
- ❌ Direct mutation of state arrays (use spread operator or concat)
- ❌ Calling storage.setExpenses() on every keystroke (debounce)
- ❌ Rendering full expense list without virtualization (kills perf @ 1000+)
- ❌ Hardcoding category list (use constants.js, make extensible)
- ❌ Skipping error boundaries (wrap in try/catch, show fallback UI)

**Questions for Clarification:**
- Should deleted expenses go to archive or true delete? (Recommend soft-delete)
- Multi-month budget or monthly budget? (Architecture assumes monthly; changeable)
- Dark mode support in Phase 1 or Phase 3? (Recommend Phase 3)
