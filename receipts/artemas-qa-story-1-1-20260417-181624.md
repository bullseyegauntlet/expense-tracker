- ✅ **AC2: User can select a category from presets (Food, Transport, Rent, Entertainment, Other)**

  - Defaults to system date on form load

- ✅ **AC4: User can add an optional note**
  - `<textarea>` field, not required
  - Stored in expense.note; displays as "—" if empty in list

- ✅ **AC5: Clicking "Add" validates amount > 0 and category is selected**
  - Client-side validation in `AddExpenseForm.handleSubmit()`
  - Error message displayed: "Amount must be greater than 0"
  - Category validation also present (catches empty category)
  - Form submission blocked until valid

- ✅ **AC6: On success, expense appears in the list and dashboard updates**
  - New expense added via `addExpense()` hook
  - State updates trigger re-render
  - Dashboard re-calculates `getTotalByCategory()` immediately
  - ExpenseList displays expense at top (prepended to array)

- ✅ **AC7: Form clears or closes after successful submission**
  - Form clears all fields on success:
    - amount → ''
    - category → 'Food'
    - date → today's date
    - note → ''
    - error → ''
  - Ready for next entry immediately

- ✅ **AC8: Submit completes in <5 seconds**
  - Actual submission: <100ms (no network calls)
